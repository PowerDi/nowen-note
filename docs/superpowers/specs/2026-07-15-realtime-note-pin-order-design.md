# 树形与目录列表置顶实时重排设计

## 目标

无论用户从编辑器顶部工具栏、树形笔记列表还是目录笔记列表切换置顶状态，相关界面都在服务端更新成功后立即同步，并按置顶优先规则重新排列，无需刷新页面。

## 排序规则

- 所有置顶笔记形成列表顶部区域。
- 置顶区域内部继续使用当前排序规则。
- 普通区域内部继续使用当前排序规则。
- 手动排序时，两个区域内部均按 `sortOrder` 排列；取消置顶后笔记按原 `sortOrder` 回到普通区域。
- 搜索视图保留原相关性顺序，只按置顶状态稳定分组。
- 非手动排序继续按标题、创建时间或更新时间及当前方向排列。

## 架构与数据流

新增一个共享的纯排序辅助函数，负责“置顶优先 + 区域内比较器”。目录列表和树形列表都调用该函数，避免两处规则继续分叉。

树形笔记列表使用独立的 `notesByNotebookId` 缓存。增加只同步 `isPinned` 的缓存协调逻辑：当全局 `state.notes` 中的置顶状态变化时，更新树形缓存中已存在的对应笔记，并保持无变化时的引用稳定。这样从编辑器工具栏或目录列表触发的更新可以立即传入树形列表。

各入口的数据流如下：

1. 调用现有笔记更新 API。
2. API 成功后更新全局目录列表状态。
3. 若目标是当前笔记，同步当前笔记状态，使顶部工具栏图标立即变化。
4. 树形缓存接收全局置顶状态变化。
5. 两种列表通过共享排序规则重新派生展示顺序。

树形列表自身触发置顶时继续直接更新树形缓存和全局列表；当前笔记仍同步到编辑器状态。

## 错误处理

沿用现有的服务端确认后更新策略，不增加乐观更新。API 失败时不修改本地置顶状态，由现有错误处理路径提示失败，避免界面与服务端状态不一致。
